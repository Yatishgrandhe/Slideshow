Performing Calculations in Python

Python Math Operators

Python can perform all the mathematical operations you learned in school, plus some additional ones that are useful in programming.

Basic Arithmetic Operators:

  • Addition (+): Adds two numbers together
  • Subtraction (-): Subtracts the second number from the first
  • Multiplication (*): Multiplies two numbers
  • Division (/): Divides the first number by the second (gives decimal result)
  • Floor Division (//): Divides and rounds down to the nearest whole number
  • Modulo (%): Gives the remainder after division
  • Exponentiation (**): Raises a number to a power

Order of Operations:

Python follows the same math rules you learned in school:

  1. Parentheses: Do calculations inside () first
  2. Exponents: Calculate powers (**)
  3. Multiplication/Division: From left to right
  4. Addition/Subtraction: From left to right
Math Operation Examples
# Basic arithmetic
a = 10
b = 3

print(a + b)   # 13 (addition)
print(a - b)   # 7 (subtraction)
print(a * b)   # 30 (multiplication)
print(a / b)   # 3.333... (division)
print(a // b)  # 3 (floor division)
print(a % b)   # 1 (remainder)
print(a ** b)  # 1000 (10 to the power of 3)

# Order of operations
result = 2 + 3 * 4
print(result)  # 14 (multiply first, then add)

# Using parentheses to control order
result = (2 + 3) * 4
print(result)  # 20

# Working with variables
price = 19.99
quantity = 3
total = price * quantity
print(f"Total: ${total:.2f}")

# Complex calculations
radius = 5
pi = 3.14159
area = pi * radius ** 2
print(f"Area: {area:.2f}")

Understanding Integer and Float Types

Number Types in Python

Python has different types of numbers, each with specific characteristics and use cases.

Integer (int):

  • Whole numbers: 1, 100, -5, 0
  • No decimal point: Used for counting, indexing
  • Unlimited precision: Can handle very large numbers
  • Memory efficient: Takes less space than floats

Float (float):

  • Decimal numbers: 3.14, -2.5, 0.0
  • Scientific notation: 1.5e3 = 1500.0
  • Limited precision: May have rounding errors
  • Used for: Calculations requiring decimals

Type Conversion:

  • int(): Convert to integer (truncates decimal)
  • float(): Convert to float
  • Automatic conversion: Python converts when needed
  • Explicit conversion: Better to be explicit

Using Variables for Calculations

Storing Calculation Results

Variables are essential for storing intermediate results and making calculations more readable and reusable.

Why Use Variables in Math:

  • Readability: Makes code easier to understand
  • Reusability: Use the same value multiple times
  • Debugging: Easier to check intermediate results
  • Maintainability: Change values in one place

Best Practices:

  • Update carefully: Be careful when modifying variables
  • Check types: Ensure you're using the right data types

Common Patterns:

  • Accumulators: Variables that collect values
  • Counters: Variables that track numbers
  • Temporary storage: Store intermediate results
  • Constants: Values that don't change

üî¢ Day 1, Slide 5: Order of Operations

Understanding Python's Calculation Priority

PEMDAS in Python

Python follows the standard mathematical order of operations, just like you learned in school.

Order of Operations (PEMDAS):

  1. P - Parentheses: () - Highest priority
  2. E - Exponents: ** - Second priority
  3. M - Multiplication: * - Third priority
  4. D - Division: /, // - Third priority
  5. A - Addition: + - Fourth priority
  6. S - Subtraction: - - Fourth priority

Important Rules:

  • Left to right: For same priority operations
  • Parentheses override: Always calculate inside () first
  • Be explicit: Use parentheses when unsure
  • Readability: Complex expressions benefit from parentheses

Common Mistakes:

  • Forgetting parentheses for intended order
  • Assuming left-to-right for all operations
  • Mixing different types of division
  • Not considering operator precedence
Order of Operations Examples
# Basic PEMDAS examples
result = 2 + 3 * 4
print(result)  # 14 (not 20)

result = (2 + 3) * 4
print(result)  # 20

# Exponents have higher priority
result = 2 ** 3 + 1
print(result)  # 9 (not 16)

# Division and multiplication have same priority
result = 10 / 2 * 3
print(result)  # 15.0 (left to right)

# Complex expression
result = 2 + 3 * 4 ** 2 - 6 / 2
print(result)  # 47.0

# Breaking it down:
# 1. 4 ** 2 = 16
# 2. 3 * 16 = 48
# 3. 6 / 2 = 3
# 4. 2 + 48 - 3 = 47

# Using parentheses for clarity
result = (2 + 3) * (4 ** 2) - (6 / 2)
print(result)  # 77.0

# Common mistake: forgetting parentheses
# Wrong way
result = 1 / 2 + 1
print(result)  # 1.5

# Right way (if you want 1/(2+1))
result = 1 / (2 + 1)
print(result)  # 0.333...

# Practical example: compound interest
principal = 1000
rate = 0.05
time = 2
amount = principal * (1 + rate) ** time
print(f"Amount: ${amount:.2f}")

# Temperature conversion
celsius = 25
fahrenheit = celsius * 9/5 + 32
print(f"{celsius}¬∞C = {fahrenheit}¬∞F")

Handling Type Conversions in Calculations

Type Conversion in Math

Python automatically converts between data types during calculations, but understanding this process helps avoid errors.

Automatic Type Conversion:

  • int + float: Result is float
  • int / int: Result is float (in Python 3)
  • int // int: Result is int (floor division)
  • Mixed operations: Python chooses the more precise type

Explicit Type Conversion:

  • int(): Convert to integer (truncates decimal)
  • float(): Convert to float
  • round(): Round to nearest integer
  • abs(): Get absolute value

Common Issues:

  • String + number causes TypeError
  • Division by zero causes ZeroDivisionError
  • Floating point precision errors
  • Overflow with very large numbers
Type Conversion Examples

print(type(result))  # 

# Floor division returns int
result = 10 // 3
print(result)  # 3
print(type(result))  # 

# Explicit type conversion

integer = int(decimal)
print(integer)  # 3

whole = 5
decimal_num = float(whole)
print(decimal_num)  # 5.0

# Rounding

rounded = round(number)
print(rounded)  # 4

# Absolute value

positive = abs(negative)
print(positive)  # 5

# Common error: string + number
# This will cause TypeError:
# result = "5" + 3

# Fix: convert string to number

# Division by zero error
# This will cause ZeroDivisionError:
# result = 10 / 0

    result = "Cannot divide by zero"

üêõ Day 1, Slide 9: Math Debugging

Finding and Fixing Math Errors

Common Math Errors

Understanding common math errors helps you write more robust code and debug problems quickly.

Type Errors:

  • String + Number: TypeError when adding incompatible types
  • Wrong data types: Using strings in calculations
  • Type conversion: Converting invalid strings to numbers

Value Errors:

  • Division by zero: ZeroDivisionError
  • Invalid conversions: ValueError when converting strings
  • Overflow: Numbers too large for memory

Precision Errors:

  • Floating point: 0.1 + 0.2 ‚â† 0.3
  • Rounding errors: Accumulated precision loss
  • Comparison issues: Using == with floats
Debugging Examples
# Type Error examples
# This will cause TypeError:
# result = "5" + 3

# Fix: convert string to number
result = int("5") + 3
print(result)  # 8

# Division by zero error
# This will cause ZeroDivisionError:
# result = 10 / 0

# Fix: check before dividing
def safe_divide(numerator, denominator):
    if denominator == 0:
        return "Cannot divide by zero"
    return numerator / denominator

print(safe_divide(10, 2))   # 5.0
print(safe_divide(10, 0))   # Cannot divide by zero

# Value Error examples
# This will cause ValueError:
# number = int("abc")

# Fix: use try-except
def safe_convert_to_int(value):
    try:
        return int(value)
    except ValueError:
        return f"Cannot convert '{value}' to integer"

print(safe_convert_to_int("123"))  # 123
print(safe_convert_to_int("abc"))  # Cannot convert 'abc' to integer

# Floating point precision error
result = 0.1 + 0.2
print(result)  # 0.30000000000000004

# Fix: use round() for comparison
if round(0.1 + 0.2, 1) == 0.3:
    print("Equal!")
else:
    print("Not equal!")

# Better fix: use decimal module
from decimal import Decimal
result = Decimal('0.1') + Decimal('0.2')
print(result)  # 0.3

# Debugging tips
def debug_calculation(a, b, operation):
    print(f"a = {a} (type: {type(a)})")
    print(f"b = {b} (type: {type(b)})")
    print(f"operation = {operation}")
    
    if operation == '+':
        result = a + b
    elif operation == '*':
        result = a * b
    elif operation == '/':
        if b == 0:
            result = "Error: Division by zero"
        else:
            result = a / b
    else:
        result = "Unknown operation"
    
    print(f"result = {result}")
    return result

# Test the debug function
debug_calculation(5, 3, '+')
debug_calculation(10, 0, '/')
