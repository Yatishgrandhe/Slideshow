üîß Day 1, Slide 1: Functions Basics

Creating Reusable Code Blocks

What are Functions?

Functions are reusable blocks of code that perform specific tasks. They help organize code and avoid repetition.

Function Declaration:

  • def keyword: Used to define a function
  • Function name: Choose descriptive names
  • Parameters: Input values the function receives
  • Function body: Code that runs when function is called
  • Return statement: Output value the function sends back

Why Use Functions?

  • Reusability: Write once, use many times
  • Organization: Break code into logical pieces
  • Maintainability: Easier to fix and update
  • Testing: Test individual pieces of code
Function Examples
# Function declaration
def greet(name):
    return f"Hello, {name}!"

# Function call
message = greet("Alice")
print(message)  # Hello, Alice!

# Function with multiple parameters
def add(a, b):
    return a + b

result = add(5, 3)
print(result)  # 8

# Function without parameters
def say_hello():
    print("Hello, World!")

say_hello()  # Hello, World!

# Function with no return value
def print_info(name, age):
    print(f"Name: {name}")
    print(f"Age: {age}")

print_info("Bob", 30)
# Output:
# Name: Bob
# Age: 30

# Function with default parameter
def greet_with_title(name, title="Mr."):
    return f"Hello, {title} {name}!"

print(greet_with_title("Smith"))  # Hello, Mr. Smith!
print(greet_with_title("Johnson", "Dr."))  # Hello, Dr. Johnson!

üèóÔ∏è Day 1, Slide 2: Function Components

Understanding Function Structure

Function Anatomy

Every function has specific components that work together to create reusable code.

Function Header:

  • def keyword: Tells Python this is a function
  • Function name: How you call the function
  • Parameters: Variables that receive input values
  • Colon: Marks the end of the header

Function Body:

  • Indented code: All code inside the function
  • Logic: The actual work the function does
  • Variables: Can create local variables
  • Control flow: Can use if, loops, etc.

Return Statement:

  • return keyword: Sends value back to caller
  • Optional: Functions can have no return
  • Multiple returns: Can have multiple return statements
  • Early exit: Return can exit function early
Function Component Examples
# Function with all components
def calculate_area(length, width):  # Header with parameters
    area = length * width           # Body: calculation
    return area                     # Return statement

# Function with multiple returns
def check_number(num):
    if num > 0:
        return "Positive"           # Early return
    elif num < 0:
        return "Negative"           # Early return
    else:
        return "Zero"               # Default return

# Function with no return (returns None)
def print_square(n):
    result = n ** 2
    print(f"{n} squared is {result}")
    # No return statement

# Function with local variables
def calculate_tax(income, rate):
    taxable_income = income - 10000  # Local variable
    tax = taxable_income * rate       # Local variable
    return tax

# Function with multiple calculations
def process_data(x, y, z):
    # Multiple local variables
    sum_xy = x + y
    product_yz = y * z
    average = (x + y + z) / 3
    
    return {
        'sum_xy': sum_xy,
        'product_yz': product_yz,
        'average': average
    }

# Testing the functions
print(calculate_area(5, 3))        # 15
print(check_number(5))             # Positive
print_square(4)                    # 4 squared is 16
print(calculate_tax(50000, 0.2))   # 8000.0
print(process_data(1, 2, 3))       # Dictionary with results

üìù Day 1, Slide 3: Function Naming

Best Practices for Function Names

Function Naming Conventions

Good function names make your code more readable and maintainable. Follow these conventions for professional code.

Naming Rules:

  • snake_case: Use lowercase with underscores
  • Descriptive: Name should describe what it does
  • Verbs: Start with action words
  • Consistent: Use same style throughout

Good vs Bad Names:

  • Good: calculate_area, get_user_info, validate_email
  • Bad: calc, func1, x, do_stuff
  • Specific: calculate_circle_area vs calculate_area
  • Clear: is_valid_email vs check_email

Naming Patterns:

  • get_*: Retrieve data (get_user_name)
  • set_*: Set values (set_user_age)
  • is_*: Boolean checks (is_valid)
  • has_*: Boolean checks (has_permission)
  • calculate_*: Math operations (calculate_total)
Function Naming Examples
# Good function names
def calculate_area(length, width):
    return length * width

def get_user_name(user_id):
    return f"User {user_id}"

def is_valid_email(email):
    return '@' in email and '.' in email

def has_permission(user, action):
    return user.can_perform(action)

def set_user_age(user, age):
    user.age = age

# Bad function names (avoid these)
def calc(a, b):  # Too short, unclear
    return a * b

def func1():     # Not descriptive
    print("Hello")

def x(y):        # Meaningless names
    return y * 2

def do_stuff():  # Vague description
    pass

# Better naming examples
def calculate_circle_area(radius):
    import math
    return math.pi * radius ** 2

def calculate_rectangle_area(length, width):
    return length * width

def validate_user_input(input_data):
    return len(input_data) > 0

def format_currency(amount):
    return f"${amount:.2f}"

def convert_temperature(celsius):
    return celsius * 9/5 + 32

# Testing the well-named functions
print(calculate_circle_area(5))      # 78.54...
print(calculate_rectangle_area(4, 6)) # 24
print(validate_user_input("hello"))   # True
print(format_currency(19.99))        # $19.99
print(convert_temperature(25))       # 77.0

üìö Day 1, Slide 4: Function Documentation

Writing Clear Function Documentation

Docstrings and Comments

Good documentation helps others (and yourself) understand what your functions do and how to use them.

Docstrings:

  • Triple quotes: """ or ''' for multi-line strings
  • First line: Brief description of what function does
  • Parameters: Describe each parameter and its type
  • Returns: Describe what the function returns
  • Examples: Show how to use the function

Accessing Documentation:

  • help(): help(function_name)
  • __doc__: function_name.__doc__
  • IDE support: Most IDEs show docstrings
  • Documentation generators: Tools like Sphinx

Best Practices:

  • Write docstrings for all functions
  • Keep them up to date with code changes
  • Use clear, concise language
  • Include examples for complex functions
Documentation Examples
def calculate_area(length, width):
    """
    Calculate the area of a rectangle.
    
    Args:
        length (float): The length of the rectangle
        width (float): The width of the rectangle
        
    Returns:
        float: The area of the rectangle
        
    Examples:
        >>> calculate_area(5, 3)
        15.0
        >>> calculate_area(2.5, 4)
        10.0
    """
    return length * width

def validate_email(email):
    """
    Check if an email address is valid.
    
    Args:
        email (str): The email address to validate
        
    Returns:
        bool: True if email is valid, False otherwise
        
    Examples:
        >>> validate_email("user@example.com")
        True
        >>> validate_email("invalid-email")
        False
    """
    return '@' in email and '.' in email

def format_currency(amount, currency="USD"):
    """
    Format a number as currency.
    
    Args:
        amount (float): The amount to format
        currency (str): The currency code (default: "USD")
        
    Returns:
        str: Formatted currency string
        
    Examples:
        >>> format_currency(19.99)
        '$19.99'
        >>> format_currency(100, "EUR")
        '‚Ç¨100.00'
    """
    if currency == "USD":
        return f"${amount:.2f}"
    elif currency == "EUR":
        return f"‚Ç¨{amount:.2f}"
    else:
        return f"{amount:.2f} {currency}"

# Accessing documentation

# Using help()
# help(calculate_area)

# Testing the documented functions

print(validate_email("test@example.com"))  # True

üéØ Day 1, Slide 5: Function Scope

Understanding Variable Scope

Variable Scope in Functions

Scope determines where variables can be accessed. Understanding scope prevents errors and helps write better code.

Local Scope:

  • Function variables: Created inside functions
  • Local access: Only accessible within the function
  • Parameter variables: Also have local scope
  • Destroyed after: Variables disappear when function ends

Global Scope:

  • Module variables: Created at file level
  • Global access: Accessible throughout the module
  • global keyword: Modify global variables from function
  • Built-in names: print, len, etc. are global

Scope Rules:

  • Local variables hide global variables
  • Functions can read global variables
  • Functions can't modify global variables without global keyword
  • Nested functions have access to outer function variables
Scope Examples
# Global variable
global_counter = 0

def increment_counter():
    """Increment the global counter."""
    global global_counter  # Declare we want to modify global
    global_counter += 1
    print(f"Counter: {global_counter}")

def create_local_variable():
    """Create a local variable."""
    local_var = "I'm local"
    print(local_var)
    # local_var is destroyed when function ends

def demonstrate_scope():
    """Show how scope works."""
    x = 10  # Local variable
    
    def inner_function():
        y = 20  # Local to inner_function
        print(f"x from outer: {x}")  # Can access outer x
        print(f"y from inner: {y}")  # Can access inner y
    
    inner_function()
    # print(y)  # Error: y is not accessible here

# Testing scope
print(f"Global counter: {global_counter}")  # 0
increment_counter()  # Counter: 1
increment_counter()  # Counter: 2

create_local_variable()  # I'm local
# print(local_var)  # Error: local_var doesn't exist here

demonstrate_scope()
# Output:
# x from outer: 10
# y from inner: 20

# Example with variable hiding
global_var = "I'm global"

def test_scope():
    global_var = "I'm local"  # Hides global variable
    print(global_var)  # I'm local

def test_global():
    global global_var  # Access global variable
    global_var = "Modified global"
    print(global_var)  # Modified global

print(global_var)  # I'm global
test_scope()       # I'm local
print(global_var)  # I'm global
test_global()      # Modified global
print(global_var)  # Modified global

üì• Day 1, Slide 6: Function Parameters

Passing Data to Functions

Types of Parameters

Parameters allow functions to receive data and make them more flexible and reusable.

Required Parameters:

  • Positional: Must be provided in order
  • No defaults: Function won't work without them
  • Clear purpose: Essential for function operation

Default Parameters:

  • Optional: Have default values
  • Can be omitted: Function uses default if not provided
  • Must come last: After required parameters
  • Mutable defaults: Be careful with lists/dicts

Parameter Types:

  • Positional: Passed by position
  • Keyword: Passed by name
  • Arbitrary: *args for variable number
  • Keyword arbitrary: **kwargs for named arguments
Parameter Examples
# Required parameters
def greet(name, age):
    return f"Hello {name}, you are {age} years old"

print(greet("Alice", 25))  # Hello Alice, you are 25 years old

# Default parameters
def greet_with_title(name, title="Mr.", greeting="Hello"):
    return f"{greeting} {title} {name}!"

print(greet_with_title("Smith"))                    # Hello Mr. Smith!
print(greet_with_title("Johnson", "Dr."))          # Hello Dr. Johnson!
print(greet_with_title("Brown", "Ms.", "Goodbye")) # Goodbye Ms. Brown!

# Keyword arguments
def create_profile(name, age, city, country="USA"):
    return f"Name: {name}, Age: {age}, City: {city}, Country: {country}"

# Using keyword arguments (order doesn't matter)
profile1 = create_profile(name="Alice", age=25, city="New York")
profile2 = create_profile(city="London", name="Bob", age=30, country="UK")

print(profile1)  # Name: Alice, Age: 25, City: New York, Country: USA
print(profile2)  # Name: Bob, Age: 30, City: London, Country: UK

# Arbitrary arguments (*args)
def sum_all(*numbers):
    return sum(numbers)

print(sum_all(1, 2, 3))        # 6
print(sum_all(1, 2, 3, 4, 5))  # 15

# Keyword arbitrary arguments (**kwargs)
def create_person(**attributes):
    return attributes

person = create_person(name="Alice", age=25, city="New York", job="Engineer")
print(person)  # {'name': 'Alice', 'age': 25, 'city': 'New York', 'job': 'Engineer'}

# Mixed parameter types
def complex_function(required, optional="default", *args, **kwargs):
    print(f"Required: {required}")
    print(f"Optional: {optional}")
    print(f"Args: {args}")
    print(f"Kwargs: {kwargs}")

complex_function("hello", "world", 1, 2, 3, name="Alice", age=25)
# Output:
# Required: hello
# Optional: world
# Args: (1, 2, 3)
# Kwargs: {'name': 'Alice', 'age': 25}

üì§ Day 1, Slide 7: Return Values

Getting Data Back from Functions

Returning Data from Functions

Return values allow functions to send data back to the calling code, making them useful for calculations and data processing.

Return Statement:

  • return keyword: Sends value back to caller
  • Immediate exit: Function stops executing
  • Any data type: Can return numbers, strings, lists, etc.
  • Multiple values: Can return tuples, lists, dictionaries

Return Types:

  • Single value: return 42
  • Multiple values: return x, y, z
  • None: Functions without return return None
  • Complex data: Lists, dictionaries, objects

Best Practices:

  • Return early for simple cases
  • Use descriptive return values
  • Be consistent with return types
  • Document what your function returns
Return Value Examples
# Single return value
def square(number):
    return number ** 2

result = square(5)
print(result)  # 25

# Multiple return values (as tuple)
def get_name_and_age():
    return "Alice", 25

name, age = get_name_and_age()
print(f"Name: {name}, Age: {age}")  # Name: Alice, Age: 25

# Return different types based on conditions
def process_number(num):
    if num > 0:
        return "positive"
    elif num < 0:
        return "negative"
    else:
        return "zero"

print(process_number(5))   # positive
print(process_number(-3))  # negative
print(process_number(0))   # zero

# Return complex data structures
def get_user_info(user_id):
    return {
        "id": user_id,
        "name": "Alice",
        "email": "alice@example.com",
        "age": 25
    }

user = get_user_info(123)
print(user["name"])  # Alice

# Early return for simple cases
def is_even(number):
    if number % 2 == 0:
        return True
    return False

# Same as above, but more concise
def is_even_concise(number):
    return number % 2 == 0

print(is_even(4))      # True
print(is_even(7))      # False

# Function with no return (returns None)
def print_greeting(name):
    print(f"Hello, {name}!")

result = print_greeting("Alice")
print(result)  # None

# Return with calculations
def calculate_stats(numbers):
    total = sum(numbers)
    count = len(numbers)
    average = total / count
    maximum = max(numbers)
    minimum = min(numbers)
    
    return {
        "total": total,
        "count": count,
        "average": average,
        "max": maximum,
        "min": minimum
    }

stats = calculate_stats([1, 2, 3, 4, 5])
print(f"Average: {stats['average']}")  # Average: 3.0

üêõ Day 1, Slide 9: Function Debugging

Finding and Fixing Function Errors

Common Function Errors

Understanding common function errors helps you write more robust code and debug problems quickly.

Syntax Errors:

  • Missing colons: After def and function body
  • Wrong indentation: Function body must be indented
  • Missing parentheses: Around function calls
  • Unmatched quotes: In docstrings or strings

Runtime Errors:

  • TypeError: Wrong number or type of arguments
  • NameError: Undefined variables in function
  • AttributeError: Calling methods on wrong objects
  • ZeroDivisionError: Division by zero

Logic Errors:

  • Wrong return value: Function returns unexpected result
  • Side effects: Function modifies global state unexpectedly
  • Infinite loops: Function never returns
  • Off-by-one errors: Common in loops and indexing
Debugging Examples
# Syntax Error examples
# def greet(name)  # Missing colon
#     print(f"Hello {name}")

# Fixed:
def greet(name):
    print(f"Hello {name}")

# TypeError examples
def add_numbers(a, b):
    return a + b

# This will cause TypeError:
# add_numbers("hello", 5)

# Fix: add type checking
def add_numbers_safe(a, b):
    if isinstance(a, (int, float)) and isinstance(b, (int, float)):
        return a + b
    else:
        return "Error: Both arguments must be numbers"

# NameError examples
def calculate_area(length, width):
    # area = length * width  # This would work
    return area  # NameError: area is not defined

# Fixed:
def calculate_area_fixed(length, width):
    area = length * width
    return area

# Logic Error example
def is_adult(age):
    if age >= 18:  # This is correct
        return True
    else:
        return False

# Better version:
def is_adult_better(age):
    return age >= 18

# Debugging function
def debug_function(func, *args, **kwargs):
    """Debug a function by printing its inputs and outputs."""
    print(f"Calling {func.__name__} with args: {args}, kwargs: {kwargs}")
    
    try:
        result = func(*args, **kwargs)
        print(f"Result: {result}")
        return result
    except Exception as e:
        print(f"Error: {e}")
        return None

# Test the debug function
debug_function(add_numbers_safe, 5, 3)
debug_function(add_numbers_safe, "hello", 5)

# Common debugging patterns
def safe_divide(numerator, denominator):
    """Safely divide two numbers."""
    try:
        return numerator / denominator
    except ZeroDivisionError:
        return "Error: Division by zero"
    except TypeError:
        return "Error: Both arguments must be numbers"

print(safe_divide(10, 2))   # 5.0
print(safe_divide(10, 0))   # Error: Division by zero
print(safe_divide("10", 2)) # Error: Both arguments must be numbers
