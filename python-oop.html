Introduction to Object-Oriented Programming

What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm that organizes code into objects that contain data and code. It's a powerful way to structure complex programs.

Key Concepts:

  • Class: Blueprint for creating objects
  • Object: Instance of a class
  • Attributes: Data stored in objects
  • Methods: Functions that belong to objects

Why Use OOP?

  • Organization: Group related data and functions
  • Reusability: Create objects that can be reused
  • Maintainability: Easier to modify and extend
  • Abstraction: Hide complex details

Real-World Analogy:

  • Class: Like a car blueprint
  • Object: Like an actual car
  • Attributes: Like car properties (color, model)
  • Methods: Like car actions (start, stop, drive)
Class and Object Examples
# Defining a class
class Car:
    def __init__(self, brand, model, year):
        self.brand = brand
        self.model = model
        self.year = year
        self.is_running = False
    
    def start(self):
        self.is_running = True
        return f"{self.brand} {self.model} is starting"
    
    def stop(self):
        self.is_running = False
        return f"{self.brand} {self.model} is stopping"
    
    def get_info(self):
        return f"{self.year} {self.brand} {self.model}"

# Creating objects (instances)
my_car = Car("Toyota", "Camry", 2020)
your_car = Car("Honda", "Civic", 2019)

# Using object methods
print(my_car.get_info())  # 2020 Toyota Camry
print(my_car.start())     # Toyota Camry is starting
print(my_car.stop())      # Toyota Camry is stopping

# Accessing attributes
print(my_car.brand)       # Toyota
print(my_car.year)        # 2020

# Multiple objects
print(your_car.get_info())  # 2019 Honda Civic

üîß Day 2: Methods and Attributes

Working with Class Components

Class Components

Classes have two main components: attributes (data) and methods (functions). Understanding how to work with these is key to OOP.

Attributes:

  • Instance attributes: Belong to specific objects
  • Class attributes: Shared by all objects
  • Private attributes: Start with underscore (_)
  • Protected attributes: Start with double underscore (__)

Methods:

  • Instance methods: Work with object data
  • Class methods: Work with class data
  • Static methods: Don't need object or class data
  • Special methods: Start and end with double underscores

Method Types:

  • __init__(): Constructor method
  • Getter methods: Access attributes
  • Setter methods: Modify attributes
  • Utility methods: Perform calculations
Methods and Attributes Examples
class BankAccount:
    # Class attribute
    bank_name = "Python Bank"
    
    def __init__(self, account_holder, balance=0):
        # Instance attributes
        self.account_holder = account_holder
        self.balance = balance
        self._account_number = self._generate_account_number()
    
    def _generate_account_number(self):
        # Private method
        import random
        return f"ACC{random.randint(1000, 9999)}"
    
    def deposit(self, amount):
        if amount > 0:
            self.balance += amount
            return f"Deposited ${amount}. New balance: ${self.balance}"
        return "Invalid amount"
    
    def withdraw(self, amount):
        if 0 < amount <= self.balance:
            self.balance -= amount
            return f"Withdrew ${amount}. New balance: ${self.balance}"
        return "Insufficient funds"
    
    def get_balance(self):
        return f"Balance: ${self.balance}"
    
    def get_account_info(self):
        return f"Account: {self._account_number}, Holder: {self.account_holder}"

# Creating accounts
account1 = BankAccount("Alice", 1000)
account2 = BankAccount("Bob", 500)

# Using methods
print(account1.deposit(500))    # Deposited $500. New balance: $1500
print(account1.withdraw(200))   # Withdrew $200. New balance: $1300
print(account1.get_balance())   # Balance: $1300
print(account1.get_account_info())

# Class attribute
print(BankAccount.bank_name)    # Python Bank

üß¨ Day 3: Inheritance

Creating Class Hierarchies

What is Inheritance?

Inheritance allows a class to inherit attributes and methods from another class. It promotes code reuse and establishes relationships between classes.

Inheritance Concepts:

  • Parent class: Base class that others inherit from
  • Child class: Class that inherits from parent
  • Method overriding: Child can redefine parent methods
  • Method extension: Child can add new methods

Types of Inheritance:

  • Single inheritance: One parent class
  • Multiple inheritance: Multiple parent classes
  • Multilevel inheritance: Chain of inheritance
  • Hierarchical inheritance: Multiple children from one parent

Benefits:

  • Code reuse: Don't repeat code
  • Hierarchy: Organize related classes
  • Polymorphism: Use different classes interchangeably
  • Maintainability: Easier to update common code
Inheritance Examples
# Parent class
class Animal:
    def __init__(self, name, species):
        self.name = name
        self.species = species
    
    def make_sound(self):
        return "Some sound"
    
    def get_info(self):
        return f"{self.name} is a {self.species}"

# Child class inheriting from Animal
class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name, "Dog")
        self.breed = breed
    
    def make_sound(self):
        return "Woof!"
    
    def fetch(self):
        return f"{self.name} is fetching the ball"

# Another child class
class Cat(Animal):
    def __init__(self, name, color):
        super().__init__(name, "Cat")
        self.color = color
    
    def make_sound(self):
        return "Meow!"
    
    def climb(self):
        return f"{self.name} is climbing the tree"

# Using the classes
dog = Dog("Buddy", "Golden Retriever")
cat = Cat("Whiskers", "Orange")

print(dog.get_info())      # Buddy is a Dog
print(dog.make_sound())    # Woof!
print(dog.fetch())         # Buddy is fetching the ball

print(cat.get_info())      # Whiskers is a Cat
print(cat.make_sound())    # Meow!
print(cat.climb())         # Whiskers is climbing the tree

# Multiple inheritance example
class FlyingAnimal(Animal):
    def fly(self):
        return f"{self.name} is flying"

class Bird(FlyingAnimal):
    def __init__(self, name, wing_span):
        super().__init__(name, "Bird")
        self.wing_span = wing_span
    
    def make_sound(self):
        return "Tweet!"

üîí Day 4: Encapsulation

Protecting Data and Methods

What is Encapsulation?

Encapsulation is the bundling of data and methods that operate on that data within a single unit (class). It helps protect data from outside interference.

Access Modifiers:

  • Public: Accessible from anywhere (default)
  • Protected: Accessible within class and subclasses (_)
  • Private: Accessible only within class (__)

Benefits of Encapsulation:

  • Data hiding: Protect sensitive data
  • Controlled access: Control how data is modified
  • Validation: Ensure data integrity
  • Maintenance: Easier to change internal implementation

Getter and Setter Methods:

  • Getters: Access private attributes safely
  • Setters: Modify private attributes with validation
  • Property decorator: Python's way to create getters/setters
  • Benefits: Add validation and logging
Encapsulation Examples
class BankAccount:
    def __init__(self, account_holder, initial_balance=0):
        self.account_holder = account_holder
        self.__balance = initial_balance  # Private attribute
        self._account_number = self._generate_number()  # Protected
    
    def _generate_number(self):
        import random
        return f"ACC{random.randint(1000, 9999)}"
    
    # Getter method
    def get_balance(self):
        return self.__balance
    
    # Setter method with validation
    def set_balance(self, new_balance):
        if new_balance >= 0:
            self.__balance = new_balance
        else:
            raise ValueError("Balance cannot be negative")
    
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            return f"Deposited ${amount}"
        return "Invalid amount"
    
    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            return f"Withdrew ${amount}"
        return "Insufficient funds"

# Using property decorator
class Temperature:
    def __init__(self):
        self._celsius = 0
    
    @property
    def celsius(self):
        return self._celsius
    
    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("Temperature below absolute zero")
        self._celsius = value
    
    @property
    def fahrenheit(self):
        return (self._celsius * 9/5) + 32
    
    @fahrenheit.setter
    def fahrenheit(self, value):
        self.celsius = (value - 32) * 5/9

# Using the classes
account = BankAccount("Alice", 1000)
print(account.get_balance())  # 1000
account.deposit(500)
print(account.get_balance())  # 1500

üîÑ Day 5: Polymorphism

Using Different Classes Interchangeably

What is Polymorphism?

Polymorphism allows objects of different classes to be treated as objects of a common superclass. It enables flexibility and extensibility in code.

Types of Polymorphism:

  • Method overriding: Child classes redefine parent methods
  • Method overloading: Same method with different parameters
  • Duck typing: "If it walks like a duck, it's a duck"
  • Operator overloading: Redefine operators for custom classes

Benefits:

  • Flexibility: Use different objects interchangeably
  • Extensibility: Add new classes easily
  • Code reuse: Write generic code that works with many types
  • Maintainability: Easier to modify and extend

Common Patterns:

  • Interface-based programming
  • Plugin architectures
  • Strategy patterns
  • Factory patterns
Polymorphism Examples
# Base class
class Shape:
    def area(self):
        pass
    
    def perimeter(self):
        pass

# Different shape classes
class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height
    
    def perimeter(self):
        return 2 * (self.width + self.height)

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        import math
        return math.pi * self.radius ** 2
    
    def perimeter(self):
        import math
        return 2 * math.pi * self.radius

# Polymorphic function
def print_shape_info(shape):
    print(f"Area: {shape.area():.2f}")
    print(f"Perimeter: {shape.perimeter():.2f}")

# Using different shapes
shapes = [Rectangle(5, 3), Circle(4)]

for shape in shapes:
    print_shape_info(shape)
    print()

# Duck typing example
class Duck:
    def swim(self):
        return "Duck swimming"
    
    def fly(self):
        return "Duck flying"

class Airplane:
    def fly(self):
        return "Airplane flying"

def make_it_fly(flying_object):
    return flying_object.fly()

# Both work because they have fly method
print(make_it_fly(Duck()))      # Duck flying
print(make_it_fly(Airplane()))  # Airplane flying

# Operator overloading
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def __add__(self, other):
        return Point(self.x + other.x, self.y + other.y)
    
    def __str__(self):
        return f"Point({self.x}, {self.y})"

p1 = Point(1, 2)
p2 = Point(3, 4)
p3 = p1 + p2  # Uses __add__ method
print(p3)  # Point(4, 6)
