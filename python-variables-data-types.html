Introduction to Python Programming

What is Python?

Python is a high-level, interpreted programming language designed to be readable and simple, making it perfect for beginners.

Why Python?

  • Easy to Learn: Python reads almost like English
  • Versatile: Used in web development, data science, automation, and AI
  • Popular: One of the most in-demand programming languages
  • Beginner-Friendly: Perfect starting point for your coding journey

Where is Python Used?

  • Web Development: Building websites and web applications
  • Data Science: Analyzing data and creating visualizations
  • Automation: Automating repetitive tasks
  • Artificial Intelligence: Machine learning and AI applications
  • Game Development: Creating games and interactive applications

üìö Day 1, Slide 2: Python History

The Story Behind Python

Python's Origins

Python was created by Guido van Rossum and was first released in 1991. The name "Python" comes from Monty Python's Flying Circus, not the snake!

Key Milestones:

  • 1991: Python 1.0 released
  • 2000: Python 2.0 with new features
  • 2008: Python 3.0 (major update)
  • 2020: Python 2.7 end of life
  • Today: Python 3.x is the standard

Python Philosophy:

  • Readability: Code should be easy to read
  • Simplicity: Simple is better than complex
  • Explicit: Explicit is better than implicit
  • Beautiful: Beautiful is better than ugly

Why Python 3?

  • Better Unicode support
  • Improved print function
  • Enhanced exception handling
  • More consistent syntax
Python 2 vs Python 3
# Python 2 (old way)
print "Hello, World!"
print "Hello", "World"

# Python 3 (current way)
print("Hello, World!")
print("Hello", "World")

# Python 2 division
result = 5 / 2  # Returns 2 (integer division)

# Python 3 division  
result = 5 / 2  # Returns 2.5 (float division)
result = 5 // 2 # Returns 2 (integer division)

# Python 3 f-strings (new feature)
name = "Alice"
age = 25
print(f"{name} is {age} years old")

Writing Your First Code

Hello, World!

Every programmer's journey begins with a "Hello, World!" program. It's a tradition that dates back to the 1970s.

Creating Your First Program:

  • Step 2: Create a new file called "hello.py"

Understanding the Code:

üìù Day 1, Slide 5: Comments and Documentation

Writing Clear and Readable Code

Why Comments Matter

Comments help you and others understand your code. They're essential for writing maintainable and professional code.

Types of Comments:

  • Single-line comments: Start with #
  • Multi-line comments: Use triple quotes """
  • Inline comments: Comments on the same line as code
  • Docstrings: Documentation for functions and classes

Best Practices:

  • Explain WHY, not WHAT (code shows what)
  • Keep comments up to date with code changes
  • Use clear, concise language
  • Comment complex logic and algorithms
  • Don't over-comment obvious code

When to Use Comments:

  • Explaining complex algorithms
  • Documenting function purposes
  • Explaining business logic
  • Temporary debugging notes
  • File headers and module descriptions
Comment Examples
# This is a single-line comment
print("Hello, World!")  # This is an inline comment

"""
This is a multi-line comment.
It can span multiple lines.
Useful for longer explanations.
"""

# Calculate the area of a circle
# Formula: area = œÄ * radius¬≤

pi = 3.14159

    """
    Calculate the area of a circle.
    
    Args:
        radius (float): The radius of the circle
        
    Returns:
        float: The area of the circle
    """
    pi = 3.14159
    return pi * radius ** 2

üìã Day 1, Slide 7: Basic Syntax Rules

Understanding Python's Grammar

Python Syntax Fundamentals

Python has specific rules that make it readable and consistent. Understanding these rules is crucial for writing proper Python code.

Indentation Rules:

  • Use spaces or tabs: But be consistent (spaces recommended)
  • 4 spaces: Standard indentation (1 tab = 4 spaces)
  • Indentation matters: It defines code blocks
  • Be consistent: Don't mix tabs and spaces

Line Structure:

  • One statement per line: Usually best practice
  • Semicolons: Optional, rarely used in Python
  • Line continuation: Use \ for long lines
  • Parentheses: Can span multiple lines naturally

Naming Conventions:

  • snake_case: For variables and functions
  • PascalCase: For classes
  • UPPER_CASE: For constants
  • Descriptive names: Make code self-documenting
Syntax Examples
# Correct indentation
if x > 0:
    print("Positive")
    x = x + 1
else:
    print("Negative or zero")

# Line continuation
long_string = "This is a very long string " \
              "that spans multiple lines"

# Or using parentheses
long_string = ("This is a very long string "
               "that spans multiple lines")

# Naming conventions
user_name = "Alice"          # snake_case
MAX_ATTEMPTS = 3             # UPPER_CASE
class UserAccount:           # PascalCase
    def get_user_info(self): # snake_case
        pass

# Multiple statements (avoid this)
x = 1; y = 2; print(x + y)

# Better approach
x = 1
y = 2
print(x + y)

üêõ Day 1, Slide 8: Common Errors and Debugging

Understanding and Fixing Errors

Types of Python Errors

Errors are a normal part of programming. Understanding them helps you write better code and fix problems quickly.

SyntaxError:

  • Missing colons: After if, for, def statements
  • Missing parentheses: Around function calls
  • Unmatched quotes: String not properly closed
  • Indentation errors: Inconsistent spacing

NameError:

  • Undefined variables: Using variables before defining
  • Typos: Misspelled variable names
  • Scope issues: Variables not in current scope

TypeError:

  • Wrong data types: Adding string to number
  • Missing arguments: Function calls with wrong parameters
  • Invalid operations: Operations on incompatible types
Error Examples and Fixes
# SyntaxError examples
if x > 0    # Missing colon
    print("Positive")

# Fixed:
if x > 0:
    print("Positive")

# NameError examples
print(undefined_variable)  # Variable not defined

# Fixed:
defined_variable = "Hello"
print(defined_variable)

# TypeError examples
result = "5" + 3  # Can't add string and int

# Fixed:
result = int("5") + 3  # Convert string to int
# or
result = "5" + str(3)  # Convert int to string

# IndentationError
if x > 0:
print("Positive")  # Wrong indentation

# Fixed:
if x > 0:
    print("Positive")
