Why Comments Matter

Comments help you and others understand your code. They're essential for writing maintainable and professional code.

Types of Comments:

When to Use Comments:

  • File headers and module descriptions

üìã Day 1, Slide 7: Basic Syntax Rules

Understanding Python's Grammar

Python Syntax Fundamentals

Python has specific rules that make it readable and consistent. Understanding these rules is crucial for writing proper Python code.

Indentation Rules:

  • Use spaces or tabs: But be consistent (spaces recommended)
  • 4 spaces: Standard indentation (1 tab = 4 spaces)
  • Indentation matters: It defines code blocks
  • Be consistent: Don't mix tabs and spaces

Line Structure:

  • One statement per line: Usually best practice
  • Semicolons: Optional, rarely used in Python
  • Line continuation: Use \ for long lines
  • Parentheses: Can span multiple lines naturally

Naming Conventions:

  • snake_case: For variables and functions
  • PascalCase: For classes
  • UPPER_CASE: For constants
  • Descriptive names: Make code self-documenting
Syntax Examples
# Correct indentation
if x > 0:
    print("Positive")
    x = x + 1
else:
    print("Negative or zero")

# Line continuation
long_string = "This is a very long string " \
              "that spans multiple lines"

# Or using parentheses
long_string = ("This is a very long string "
               "that spans multiple lines")

# Naming conventions
user_name = "Alice"          # snake_case
MAX_ATTEMPTS = 3             # UPPER_CASE
class UserAccount:           # PascalCase
    def get_user_info(self): # snake_case
        pass

# Multiple statements (avoid this)
x = 1; y = 2; print(x + y)

# Better approach
x = 1
y = 2
print(x + y)

üêõ Day 1, Slide 8: Common Errors and Debugging

Understanding and Fixing Errors

Types of Python Errors

Errors are a normal part of programming. Understanding them helps you write better code and fix problems quickly.

SyntaxError:

  • Missing colons: After if, for, def statements
  • Missing parentheses: Around function calls
  • Unmatched quotes: String not properly closed
  • Indentation errors: Inconsistent spacing

NameError:

  • Undefined variables: Using variables before defining
  • Typos: Misspelled variable names
  • Scope issues: Variables not in current scope

TypeError:

  • Wrong data types: Adding string to number
  • Missing arguments: Function calls with wrong parameters
  • Invalid operations: Operations on incompatible types
Error Examples and Fixes
# SyntaxError examples
if x > 0    # Missing colon
    print("Positive")

# Fixed:
if x > 0:
    print("Positive")

# NameError examples
print(undefined_variable)  # Variable not defined

# Fixed:
defined_variable = "Hello"
print(defined_variable)

# TypeError examples
result = "5" + 3  # Can't add string and int

# Fixed:
result = int("5") + 3  # Convert string to int
# or
result = "5" + str(3)  # Convert int to string

# IndentationError
if x > 0:
print("Positive")  # Wrong indentation

# Fixed:
if x > 0:
    print("Positive")

üì¶ Slide 11: What are Variables?

Storing Data in Python

Understanding Variables

Variables are like containers that store data in your program. They allow you to save values and use them later.

What is a Variable?

  • Container: Holds data for later use
  • Name: A label you choose for the data
  • Value: The actual data stored inside
  • Dynamic: Can change during program execution

Why Use Variables?

  • Store data for reuse
  • Make code more readable
  • Perform calculations
  • Organize information
  • Make code flexible

Variable Rules:

  • Names must start with a letter or underscore
  • Can contain letters, numbers, and underscores
  • Case sensitive (age ‚â† Age)
  • Cannot use Python keywords (if, for, def, etc.)
  • Use descriptive names
Variable Examples
# Creating variables
name = "Alice"
age = 25
height = 5.6
is_student = True

# Using variables
print(f"{name} is {age} years old")
print(f"Height: {height} feet")
print(f"Is student: {is_student}")

# Changing variable values
age = 26  # Update the age
print(f"Updated age: {age}")

# Good variable names
user_name = "Bob"
user_age = 30
is_active = True

# Bad variable names (avoid these)
a = "Alice"  # Too short
x = 25       # Not descriptive
userName = "Bob"  # Inconsistent style

# Multiple assignment
first_name, last_name = "John", "Doe"
print(f"{first_name} {last_name}")

# Swapping variables
a = 1
b = 2
a, b = b, a  # Swap values
print(f"a = {a}, b = {b}")

üìù Slide 12: String Data Type

Working with Text

What are Strings?

Strings are sequences of characters (letters, numbers, symbols) used to represent text in Python.

Creating Strings:

  • Single quotes: 'Hello'
  • Double quotes: "Hello"
  • Triple quotes: '''Multi-line''' or """Multi-line"""

String Operations:

  • Slicing: Getting parts of strings

String Methods:

  • upper()/lower(): Change case
  • strip(): Remove whitespace
  • replace(): Replace text
  • split(): Break into parts
  • join(): Combine parts

üî¢ Slide 13: Numeric Data Types

Integers and Floating-Point Numbers

Numeric Types in Python

Python has two main numeric types: integers (whole numbers) and floating-point numbers (decimal numbers).

Integer (int):

  • Whole numbers: 1, 100, -5, 0
  • No decimal point: Always exact
  • Unlimited size: Can be very large
  • Operations: +, -, *, /, //, %, **

Float (float):

  • Decimal numbers: 3.14, -2.5, 0.0
  • Approximate: May have rounding errors
  • Scientific notation: 1.5e3 = 1500
  • Operations: Same as integers

Mathematical Operations:

  • Addition: +
  • Subtraction: -
  • Multiplication: *
  • Division: / (always returns float)
  • Floor division: // (returns integer)
  • Modulo: % (remainder)
  • Exponentiation: **

‚úÖ Slide 14: Boolean Data Type

True and False Values

What are Booleans?

Booleans represent logical values: True or False. They're essential for decision-making in programs.

Boolean Values:

  • True: Represents a true condition
  • False: Represents a false condition
  • Case sensitive: Must be capitalized
  • Used in: Conditions, loops, comparisons

Comparison Operators:

  • Equal: == (not =)
  • Not equal: !=
  • Greater than: >
  • Less than: <
  • Greater or equal: >=
  • Less or equal: <=

Logical Operators:

  • and: Both conditions must be True
  • or: At least one condition must be True
  • not: Reverses the boolean value
Boolean Examples
# Boolean values
is_student = True
is_working = False

print(is_student)   # True
print(is_working)   # False

# Comparison operators
age = 25
print(age == 25)    # True
print(age != 30)    # True
print(age > 20)     # True
print(age < 30)     # True
print(age >= 25)    # True
print(age <= 25)    # True

# Common mistake
x = 5
# x = 5  # This is assignment, not comparison
# x == 5 # This is comparison

print(will_get_wet)  # False

# Truthy and Falsy values
print(bool(0))      # False
print(bool(1))      # True

# Using in conditions

print(f"Is teenager: {is_teenager}")  # True

üîÑ Slide 15: Type Conversion

Converting Between Data Types

What is Type Conversion?

Type conversion (casting) is the process of converting one data type to another. This is often necessary when working with different types of data.

Conversion Functions:

  • int(): Convert to integer
  • float(): Convert to float
  • str(): Convert to string
  • bool(): Convert to boolean
  • list(): Convert to list
  • tuple(): Convert to tuple

When to Convert:

  • User input is always a string
  • Mathematical operations need numbers
  • Displaying numbers as text
  • Working with different data structures
  • Data validation and processing

Important Rules:

  • Not all conversions are possible
  • Some conversions may lose data
  • Be careful with floating-point precision
  • Always validate user input
Type Conversion Examples
# String to number
age_string = "25"
age_int = int(age_string)
print(age_int)  # 25

height_string = "5.8"
height_float = float(height_string)
print(height_float)  # 5.8

# Number to string
age = 25
age_string = str(age)
print(age_string)  # "25"

# Float to int (truncates decimal)
pi = 3.14159
pi_int = int(pi)
print(pi_int)  # 3

# Boolean conversion
print(bool(0))      # False
print(bool(1))      # True
print(bool(""))     # False
print(bool("Hello")) # True

# User input conversion
user_input = input("Enter your age: ")  # Always returns string
age = int(user_input)  # Convert to integer
print(f"You are {age} years old")

# Safe conversion with error handling
def safe_int_conversion(value):
    try:
        return int(value)
    except ValueError:
        print(f"Cannot convert '{value}' to integer")
        return None

# Examples
print(safe_int_conversion("25"))    # 25
print(safe_int_conversion("abc"))   # None

# Complex conversions
numbers = [1, 2, 3, 4, 5]
numbers_tuple = tuple(numbers)
print(numbers_tuple)  # (1, 2, 3, 4, 5)

text = "Hello"
text_list = list(text)
print(text_list)  # ['H', 'e', 'l', 'l', 'o']

üì• Slide 16: User Input

Getting Data from Users

Getting User Input

The input() function allows your program to get data from users. This makes your programs interactive and dynamic.

The input() Function:

  • Always returns a string: Even if user types a number
  • Takes a prompt: Text to show the user
  • Waits for Enter: Program pauses until user presses Enter
  • Can be stored: Save the input in a variable

Working with Input:

  • Always convert input to the right type
  • Provide clear prompts to users
  • Handle invalid input gracefully
  • Use try/except for error handling
  • Validate user input when necessary

Best Practices:

  • Give clear, specific prompts
  • Tell users what format you expect
  • Handle empty input appropriately
  • Provide helpful error messages
  • Test with different types of input
User Input Examples
# Basic input
name = input("What is your name? ")
print(f"Hello, {name}!")

# Input with conversion

age = int(age_string)

# Multiple inputs

# Input with validation
def get_valid_age():
    while True:
        try:
            age = int(input("Enter your age: "))
            if 0 <= age <= 150:
                return age
            else:
                print("Please enter a valid age (0-150)")

üö´ Slide 19: None and Special Values

Understanding None and Special Cases

What is None?

None is a special value in Python that represents "nothing" or "no value". It's used to indicate the absence of a value.

None Characteristics:

  • Unique value: There's only one None
  • Falsy: Evaluates to False in conditions
  • Common uses: Default return value, missing data
  • Type: NoneType (special type)

When to Use None:

  • Default function return value
  • Indicating missing or invalid data
  • Optional parameters
  • Initializing variables
  • Error conditions

Working with None:

  • Use is None to check for None
  • Use is not None to check for not None
  • Don't use == or != with None
  • None is not the same as 0, "", or False
None Examples

# Checking for None

    print("x has a value")

# Common mistake

    print("x is None")
