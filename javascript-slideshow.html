The Language of the Web

What is JavaScript?

JavaScript is a high-level, interpreted programming language that powers the modern web. It's the language that makes websites interactive and dynamic.

Why JavaScript?

  • Web Native: Runs in every web browser
  • Versatile: Frontend, backend, mobile apps
  • Popular: Most used programming language
  • Modern: Constantly evolving with new features

Where is JavaScript Used?

  • Web Development: Interactive websites and web apps
  • Mobile Apps: React Native, Ionic, Cordova
  • Server-Side: Node.js for backend development
  • Desktop Apps: Electron for cross-platform apps
  • Game Development: Browser-based games
JavaScript Code Example
// This is how simple JavaScript is!
console.log("Hello, World!");

// Variables are easy to understand
let name = "Alice";
let age = 25;
console.log(`${name} is ${age} years old`);

// Simple math operations
let result = 10 + 5;
console.log(`10 + 5 = ${result}`);

// Easy to read loops
for (let i = 0; i < 3; i++) {
    console.log(`Learning JavaScript is fun! (step ${i+1})`);
}

Start Coding in Minutes!

How to Run JavaScript

JavaScript runs in web browsers, so you can start coding immediately without installing anything!

Method 1: Browser Console

  • Open Browser: Chrome, Firefox, Safari, or Edge
  • Press F12: Opens Developer Tools
  • Click Console: Tab where you can type JavaScript
  • Start Coding: Type and press Enter to run

Method 2: HTML File

  • Create HTML file: Save as .html extension
  • Add script tag: <script> your code </script>
  • Open in browser: Double-click the file
  • Check console: Press F12 to see output

Method 3: Online Editors

  • CodePen: codepen.io
  • JSFiddle: jsfiddle.net
  • Replit: replit.com
  • CodeSandbox: codesandbox.io

Storing Information in JavaScript

JavaScript Variables

Variables are containers for storing data values. JavaScript has three ways to declare variables.

Variable Declaration:

  • let: Modern way, block-scoped, can be reassigned
  • const: Cannot be reassigned, block-scoped
  • var: Old way, function-scoped (avoid in modern code)

JavaScript Data Types:

  • String: Text data - "Hello", 'JavaScript'
  • Number: Numbers - 42, 3.14, -5
  • Boolean: True or false values
  • Array: Ordered collections - [1, 2, 3]
  • Object: Key-value pairs - {name: "Alice"}
  • null: Intentional absence of value
  • undefined: Variable declared but not assigned

Best Practices:

  • Use const by default
  • Use let when you need to reassign
  • Use descriptive variable names
  • Use camelCase naming convention
Variable Examples
// String variables
const name = "Alice";
let greeting = "Hello, World!";

// Number variables
const age = 25;
let temperature = -5.5;
const pi = 3.14159;

// Boolean variables
const isStudent = true;
let isRaining = false;

// Array variables
const colors = ["red", "green", "blue"];
let numbers = [1, 2, 3, 4, 5];

// Object variables
const person = {
    name: "Bob",
    age: 30,
    city: "New York"
};

// Check data types
console.log(typeof name);        // "string"
console.log(typeof age);         // "number"
console.log(typeof isStudent);   // "boolean"
console.log(typeof colors);      // "object" (arrays are objects)
console.log(typeof person);      // "object"

// Using variables
console.log(`Hello, ${name}!`);
console.log(`Age: ${age}`);
console.log(`Colors: ${colors.join(", ")}`);

Reusable Blocks of Code

What are Functions?

Functions are reusable blocks of code that perform specific tasks. They help organize code and avoid repetition.

Function Declaration:

  • function keyword: Traditional way to declare functions
  • Arrow functions: Modern, concise syntax
  • Function expressions: Functions assigned to variables

Function Components:

  • Parameters: Input values the function receives
  • Function body: Code that runs when function is called
  • Return statement: Output value the function sends back
  • Function call: How you use the function

Why Use Functions?

  • Reusability: Write once, use many times
  • Organization: Break code into logical pieces
  • Maintainability: Easier to fix and update
  • Testing: Test individual pieces of code
Function Examples
// Function declaration
function greet(name) {
    return `Hello, ${name}!`;
}

// Arrow function
const add = (a, b) => {
    return a + b;
};

// Function expression
const multiply = function(x, y) {
    return x * y;
};

// Simple arrow function (one line)
const square = x => x * x;

// Using functions
console.log(greet("Alice"));     // "Hello, Alice!"
console.log(add(5, 3));          // 8
console.log(multiply(4, 6));     // 24
console.log(square(7));          // 49

// Function with multiple parameters
function calculateArea(width, height) {
    return width * height;
}

// Function with default parameters
function greetUser(name = "Guest") {
    return `Welcome, ${name}!`;
}

console.log(calculateArea(10, 5));  // 50
console.log(greetUser("Bob"));      // "Welcome, Bob!"
console.log(greetUser());           // "Welcome, Guest!"
